Property Classification System
Universal Taxonomy for All Platforms
v4 — Feb 2026

What We're Solving

We have 47+ property type definitions spread across BSTK, RentSync, the ILS sites, and Spacelist — and they don't agree with each other. "Apartment" means something different on every platform. Rooms and basements are forced into building-level classifications they don't belong in. There's no clean way to represent a "loft apartment" vs a "studio" vs a "garden suite" without overloading a single field. And when we acquire a new platform, we start the mapping problem over again from scratch.

The solution: one canonical data model that stores listings in 7 orthogonal dimensions, with a platform mapping layer that translates to whatever each site needs. Store once, display everywhere.

The Seven Dimensions
Think of them as questions we answer about each listing.
Dimension Question It Answers Example Values Required? Cardinality
Sector What broad category? residential, commercial, recreation, other Required 1:1 — one per property
Structure Type What kind of building? house, townhouse_plex, apartment_bldg, office, retail Optional* 1:1 — one per property
Subtype What specific variation? detached, mobile, cabin, low_rise, duplex, class_a Optional 1:1 — one per property
Variant What style or refinement? loft, studio, penthouse, bungalow, garden_suite Optional 1:1 — one per property
Unit Type What are you listing? apartment, basement, room, entire, suite, office Required 1:1 — one per listing
Transaction What's the deal? rent, sale, lease Required 1:1 — one per listing
Purpose Who / what is it for? standard, student, senior, vacation, corporate Required 1:N — can have multiple
* Structure Type is optional for "unit-first" listings
Some listings are defined by the unit, not the building. When someone lists "an apartment" or "a room for rent," they don't always know or care whether it's in a low-rise or high-rise. For these unit-first listings (apartment, basement, room, office, parking), Structure Type is nullable — the unit type IS the classification. When the building info is known, it's stored as enrichment. More on this below.

Two Paths to a Listing

Not every listing starts with "what building is this?"

Unit-First
"I'm listing an apartment" — the unit IS the product
Structure optional

The renter searches for the unit type directly. Building details are background context — nice to have, not required.

Apartment Basement Room Office Parking
"2BR apartment for rent" — renter doesn't care if it's low-rise or mid-rise
"Private room available" — doesn't matter if it's in a house or townhouse
Structure-First
"I'm listing my house" — the building IS the product
Structure required

The building type defines the listing. Unit type tells you what portion is available.

Entire Main Floor Suite Floor Spot
"3BR detached house for rent" — the house IS the listing
"Main floor of duplex" — building type matters, unit type refines it
Same model, same database — just two entry points
Both paths store data in the same 7 dimensions. The only difference is whether Structure Type is required (structure-first) or nullable (unit-first). When a unit-first listing does have building info — "2BR apartment in a mid-rise" — we store it as enrichment. Nothing is lost.

Platform mappings are configuration, not code
Each platform has a mapping table that translates canonical values to its display vocabulary. When we acquire a new platform, we add a mapping table — we don't restructure the data model. The canonical system is future-proof.

Ownership Types

          

            Ownership describes how a property is held — not what the building looks like. A detached house can be freehold, condo, or co-op. A unit in a high-rise can be a condo or a rental. Ownership is tracked as a separate attribute, not as part of the taxonomy.
          


          
          

            
The "Condo Problem" — solved

            

              In the old system, "Condo" was a property type alongside "Apartment" and "House." But a condo isn't a type of building — it's a legal ownership model. A condo unit can be in a high-rise, a townhouse, or a detached house. By separating ownership from structure, we stop conflating two different questions: "what does the building look like?" and "who owns it?"
            

          


          
          

            

              
Ownership Values

              
One per property — tracked independently from all 7 taxonomy dimensions

            

            

              

                
                  

                    Value
                    Description
                    Example
                    ILS Impact
                  

                
                
                  

                    freehold
                    Owner holds both the building and the land outright. Most common for detached houses.
                    3BR detached house, owner-occupied
                    No special filter
                  

                  

                    condo
                    Owner holds the unit; common areas are shared and managed by a condo corporation. Monthly condo fees apply.
                    1BR unit in a high-rise, owned by individual investor
                    Searchable filter
                  

                  

                    coop
                    Owner holds shares in a cooperative corporation that owns the building. Board approval typically required for transactions.
                    Unit in a co-op building, shareholder-occupied
                    Searchable filter
                  

                  

                    leasehold
                    Owner holds the building but leases the land from a landowner (often First Nations, government, or institutional). Land lease has an expiry date.
                    Townhouse on leased land in West Vancouver
                    Flagged as leasehold
                  

                  

                    rental
                    Property is purpose-built rental — owned by an institution or landlord, not available for individual unit purchase.
                    Purpose-built rental apartment building
                    Default (most ILS listings)
                  

                
              

            

          


          
          

            
Why this matters for the taxonomy

            

              Because ownership is a separate attribute, the same physical building can have different ownership types across its units. A converted house might have a freehold main floor and a condo basement suite. The taxonomy describes the building; ownership describes the legal arrangement. They don't interfere with each other.

Community — Display Override

          

            A Community is not a property type, not a structure type, and not a taxonomy dimension. It's a portfolio grouping — a way for property managers to present multiple buildings under a single brand name on ILS sites. When a community exists, it changes how listings appear in search results, but it doesn't change how data is classified or stored.
          

        


        
        

          
Community bumps the building down one level in the display hierarchy.

          
The community becomes the search result card. Buildings become a sub-grouping inside it.

What Community Is Not


          

            

              
✗ Not a property type

              
You can't create a "community" listing. It's a grouping layer above listings.

            

            

              
✗ Not a structure type

              
Doesn't describe a physical building. A community contains buildings, it isn't one.

            

            

              
✗ Not a taxonomy dimension

              
Doesn't affect how a unit is classified. A unit in a community has the exact same 7 dimensions as one outside it.

            

            

              
✗ Not visible at unit level

              
A renter looking at a specific unit listing doesn't see "community." It only affects the search result layer.

            

            

              
✗ Not created by landlords

              
PM companies or platform admins set up communities. Individual landlords with one building don't use this.
