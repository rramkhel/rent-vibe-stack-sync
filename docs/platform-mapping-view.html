🌲 Why WoodCabins Is Here

WoodCabins.ca is a fictional niche platform that demonstrates how the mapping layer handles future acquisitions or partnerships without changing the canonical model.

To add a new platform, you:

  1. Decide which canonical values are visible on this platform
  2. Decide the display name for each visible value (can match canonical or use platform's own vocabulary)
  3. Add rows to the mapping.platform_structure table — no schema changes needed
  4. Everything not explicitly mapped is hidden by default

WoodCabins only cares about Cottage/Cabin subtypes. It gets full granularity for those (A-Frame, Yurt, Treehouse, Chalet) while everything else — apartments, offices, duplexes — simply doesn't exist on the platform. Same canonical data, different lens.

The Point of All This

A Yurt stored as structure_type: cottage, subtype: yurt in the canonical layer displays as "Cabin" on BSTK, "House" on Rentals.ca, "Yurt" on WoodCabins, and is invisible on Spacelist. Same record. Four completely different experiences. No data duplication, no platform-specific schemas.

The canonical model names don't matter to end users — "Multi-Unit Low-Rise" never appears in any UI. What matters is that each platform's mapping table translates it into vocabulary that makes sense for that platform's audience.
