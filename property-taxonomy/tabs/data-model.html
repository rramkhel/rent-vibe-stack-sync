Key Concepts

📊
Scope is Derived, Not Declared
Single-family vs multi-family is NOT a dimension. It's calculated from unit count. IF property.units.count == 1 → Single-family billing
IF property.units.count > 1 → Multi-family billing
🏘
Community is a Wrapper, Not a Type
"Community" exists outside the property hierarchy. It's a Listing Structure concept:
  • Unit — One unit, one listing (Sarah's basement)
  • Building — One building, multiple units (50-unit apartment)
  • Community — Multiple buildings grouped (Development of 200 townhomes)
🏠
Condo is Ownership, Not Structure
"Condo" is about ownership model, not building shape. A high-rise can be condo. A townhouse can be condo. Resolution: Condo lives in a separate Ownership/Tenure attribute, not in Structure Type.
💰
Billing is Decoupled from Classification
"Nobody should be changing how they structure their data to save on billing." — Steven
The data model should reflect reality, not billing optimization.
