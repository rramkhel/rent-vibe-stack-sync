Platform Overview

How canonical data maps to each platform's schema.

ILS
Rentals.ca
Consumer-facing, simplified

Simplified categories for renters. Maps granular subtypes to broad categories.

Enterprise
BSTK / RentSync
Full granularity, unit-based

Full dimensional model exposed. Property managers get complete control.

Commercial
Spacelist
Office, Retail, Industrial

Commercial-specific fields: sqft, NNN lease, zoning. Different schema entirely.

Niche
WoodCabins
Vacation, cabin rentals

Specialty platform. A-Frame, Yurt, Treehouse are first-class citizens here.

ILS
RentFaster
Property Category + Type

Different schema structure. Uses Property Category + types.

ILS
RentBoard / RentCanada
Simpler schemas

Simpler ILS platforms with fewer categories.

Bidirectional Mapping

Outbound: Canonical → Platform
Granular canonical data maps to simplified platform display. Lossless in terms of what we store, simplified for what we show.
Inbound: Platform → Canonical
Can be "lossy" — if BSTK sends "House", we can't know if it's detached, cabin, or yurt. We store what we receive and infer where possible.
