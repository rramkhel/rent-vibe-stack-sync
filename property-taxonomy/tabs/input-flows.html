How We Get Data From Users

Three input strategies, same canonical output. Match the experience to the user's complexity tolerance.

The Macro Solution

Sarah picks "Basement" from a simple list. We populate 6+ fields. She answered 1 question.

1
Inference from Minimal Input
For: Small landlords on Rentals.ca

User answers 3-5 simple questions. We infer the rest from their answers using predetermined rules.

Example: Sarah selects "Basement"

User input: "Basement"
    → Sector = Residential (obvious)
    → Scope = Unit (renting part of something)
    → StructureType = House (most basements are in houses)
    → UnitType = Basement Suite (what she told us)
    → ListingType = For Rent (default on Rentals.ca)
    → Purpose = Conventional (default)

Result: Sarah answered 1 question. We populated 6 fields.

2
AI / Photo Analysis
For: Anyone with photos and a sentence

User uploads photos and/or types a description. AI extracts structure, amenities, and features.

Example: Sarah uploads 5 photos + types a sentence

User input: 5 photos + "Bright 1BR basement with separate entrance, close to TTC"

AI extracts from photos:
    • Hardwood floors
    • Separate entrance visible
    • In-unit laundry
    • Natural light
    • Window placement suggests house (not apartment)

AI extracts from text:
    • 1BR
    • Basement
    • Separate entrance
    • Near transit

Canonical output:
    UnitType = Basement Suite
    StructureType = House
    Bedrooms = 1
    Amenities = [Hardwood, Separate Entrance, In-unit Laundry, Near Transit]

Result: Sarah uploaded photos and wrote one sentence. We built 80% of her listing.

3

Why this is okay: Greenfield Properties managing 150 units across 3 buildings WANTS to specify "High Rise vs Mid Rise". They need community groupings and unit type breakdowns. The complexity is a feature, not a bug.
